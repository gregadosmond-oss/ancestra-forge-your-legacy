# Ancestra Crest Style Guide

The official visual style for all customer crests generated by Ancestra.
Reference image: `ancestra-crest-3d.png`

## Visual Principles

- **Photorealistic 3D rendering** — not flat illustration
- **Polished antique gold** as the primary metallic material
- **Rich heraldic color accents** inside the shield — royal blue, crimson red, deep green (based on family heritage)
- **Dimensional depth** — raised relief, cast shadows, specular highlights
- **Symmetrical composition** — centered shield with mirrored supporters
- **Ornate baroque scrollwork** — acanthus leaves, flourishes, cartouche framing

## Required Elements

1. **Crest (top)** — an animal symbol specific to the family (lion, eagle, stag, bear, wolf, dragon, falcon)
2. **Crown / Torse** — royal crown with helm below (or just torse for non-noble families)
3. **Knight's Helmet** — medieval tournament helm, facing forward
4. **Supporters (both sides)** — rampant animals flanking the shield (lions default; swap based on family lore)
5. **Central Shield** — quartered design with 4 symbols:
   - Fleur-de-lis (heritage / lineage)
   - Family initial monogram (letter in ornate serif)
   - Symbol of family trade/role (tree = growth, sword = warrior, anchor = sailor, hammer = builder, wheat = farmer, book = scholar)
   - Fleur-de-lis (repeat for balance)
6. **Banner (bottom)** — silver or gold scrolled ribbon with FAMILY SURNAME in bold gold serif capitals
7. **Below banner** — small elegant text: "SINCE [YEAR]" where year is the earliest documented ancestor

## Color Palette

- Background: textured charcoal/gray (not pure black — allows the gold to pop without losing atmosphere)
- Primary metal: polished antique gold (#C9A64C highlights, #6B4A14 shadows)
- Shield interior: royal blue (#1E3A8A) + crimson red (#991B1B) + optional deep green (#064E3B)
- Banner: aged silver (#C0C0C0) or cream with dark inscription
- Accent: ivory / off-white for text legibility

## Master Prompt Template

Use this exact structure when calling image generation APIs. Replace variables in `{{brackets}}` with user-specific data pulled from the user's Legacy Journey input.

```
Generate a photorealistic 3D heraldic coat of arms for the {{SURNAME}} family.

CREST (TOP): A gold rampant {{CREST_ANIMAL}} holding {{CREST_OBJECT}}
Below the crest: A royal crown with a medieval knight's helmet facing forward

SUPPORTERS: Two gold rampant {{SUPPORTER_ANIMAL}}s flanking the shield, standing on their hind legs

CENTRAL SHIELD (quartered): 
- Top-left: Gold fleur-de-lis on royal blue
- Top-right: Ornate gold "{{FAMILY_INITIAL}}" monogram on crimson red
- Bottom-left: Gold {{TRADE_SYMBOL}} on crimson red
- Bottom-right: Gold fleur-de-lis on royal blue

BANNER (bottom): Silver scrolled ribbon with "{{SURNAME}}" in bold gold serif capital letters
BELOW BANNER: Small elegant gold text: "SINCE {{EARLIEST_YEAR}}"

STYLE: Hyper-realistic 3D rendering, polished antique gold with deep shadows and bright specular highlights, dramatic cinematic lighting, rich dimensional depth, ornate baroque heraldic design, museum-quality luxury feel. Textured charcoal gray background.

Square 1:1 aspect ratio. Centered composition.
```

## Variable Mapping (from user inputs)

| Variable | Source | Default |
|---|---|---|
| `SURNAME` | User input (Stop 1) | — |
| `FAMILY_INITIAL` | First letter of surname | — |
| `EARLIEST_YEAR` | FamilySearch API earliest ancestor record | 1600 |
| `CREST_ANIMAL` | Claude API — based on family role | lion |
| `CREST_OBJECT` | Claude API — based on family role | sword |
| `SUPPORTER_ANIMAL` | Claude API — based on family region | lion |
| `TRADE_SYMBOL` | Claude API — based on ancestral role | oak tree |

## Example (Osmond family)

- Crest animal: **eagle** (protector / watcher — from "os" = god, "mund" = protector)
- Supporters: **lions** (Dorset, England heritage)
- Trade symbol: **oak tree** (land stewards, Haywards)
- Banner: **OSMOND**
- Since year: **1066** (Domesday Book entry) or **1688** (documented coat of arms date)

## Implementation Notes

- Keep the prompt under 1000 chars to fit API limits
- DALL-E 3 follows this structure reliably; Midjourney needs `--ar 1:1 --style raw --quality 2`
- Generate 1 image per customer; if user requests variations, generate 3 and let them pick
- Store generated crests in Lovable Cloud DB `crests` table with columns: surname, crest_url, prompt_used, generated_at, variables_json
